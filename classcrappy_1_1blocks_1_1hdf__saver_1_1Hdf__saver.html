Detailed Description

To save data efficiently in a hdf5 file.

This block is is meant to save data coming by arrays at a high rate (>1kHz). It uses the module tables from pytables. Do not forget to specify the type of data to be saved (see atom parameter) to avoid casting the data into another type, as this could result in data loss or inefficient saving

  • node: The name of the node where the data will be saved (str, default="table")
  • expected_rows: The number of expected rows in the file It is used to optimize the dumping (int, default=10**8)
  • atom: This represent the type of data to be stored in the table. It can be given as a tables.Atom instance. Also it can be given as an Numpy type or a str. (tables.Atom/numpy.dtype/str, default=tables.Int16Atom())
  • label: The key of the dict that contains the array to save. (str, default="stream")
  • metadata: A dict containing additional info to save in the hdf5 file. You can add any type of info, they will be written when opening the file at the node corresponding to its key. (dict, default={})
