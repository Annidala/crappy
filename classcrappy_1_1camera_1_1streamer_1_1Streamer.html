Detailed Description

This is a fake sensor meant to stream images that were already saved.

It needs a way to locate the time of each frame in the name of the picture, This is done using regular expressions.

init takes no args, the arguments must be given whan calling open (like all cameras).

Parameters
paththe path of the folder containing the images (str, mandatory)
patternthe regular expression matching the images and returning the time The default value is compatible with the naming method of the Camera and Videoextenso blocks. (str, default= "img_\d+_(\d+\.\d+)\.tiff"). "\d" matches digits, "\d+" matches a group of digits. () is a capturing group, returning what is inside. Dot is a special character and needs to be escaped (hence the "\.").
start_delayBefore actually streaming the image flux, you can set a delay in secongs during which the first image will be streamed in a loop. This can be useful to give time for spot seleciotn when using videoextenso. (float, default=0)
